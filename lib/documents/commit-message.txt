fix: re-inject TopAds script on ad slot rotation for container discovery

TopAds spa() only refreshes slots registered during the initial bootstrap
scan. Dynamically created containers (square02, square03) were never
registered, so spa() left them empty after rotation.

On each rotation the TopAds external script is now removed and re-injected
(same pattern used by activateDelayedAds in the SPA handler), forcing a
full DOM re-scan that discovers the new [data-topads] container. The first
mount is skipped since TopAds bootstrap handles initial fill naturally.
