fix: resolve removeChild crash in round-robin ad rotation

Ad scripts (TopAds/GPT) inject child nodes into the ad container div.
Using a React `key` swap to remount the container caused React to call
removeChild on nodes the ad script had re-parented, crashing the app.

Replace the key-based remount with an imperative DOM approach: React
owns only a stable wrapper div while the inner ad container is created
and destroyed via innerHTML/appendChild, bypassing React reconciliation.
